# rl144 Batch 3 Implementation Plan — balance, core/crust seam, terminal backend, scoreboard, aesthetics

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the game winnable (sim-gated 10–25% bot win rate), split the engine into a platform-free core with feature-gated minifb/terminal backends, add a size scoreboard + cross-backend determinism gate, then an aesthetic pass (autotiling, screens, light grading, palette, status bars).

**Architecture:** The core (rng/content/game/save/headless + a new cell-grid composer in render.rs) has zero platform calls and zero `cfg`. Its surface to a frontend is `render_cells(&Game, screen, &mut [Cell])` (80×30 cells of `{ch: u16, fg: u32, bg: u32}`) plus `Game::apply_input(u8)` and the headless entry points. Backends (`backend_minifb.rs`, `backend_term.rs`) are cfg-bounded modules selected by mutually-exclusive cargo features; `main.rs` does arg parsing + headless dispatch + backend wiring.

**Tech Stack:** Rust (MSRV 1.75, local rustc 1.97), minifb + font8x8 (both become optional, tied to `backend-minifb`), hand-rolled termios FFI for the terminal backend. Zero new crates.

### Task 3: Terminal backend (ANSI, raw termios, dirty cells)

**Files:**
- Create: `src/backend_term.rs`:
  - Termios FFI (Linux x86_64): `#[repr(C)] struct Termios {c_iflag/c_oflag/c_cflag/c_lflag: u32, c_line: u8, c_cc: [u8;32], c_ispeed/c_ospeed: u32}`; extern tcgetattr/tcsetattr/read/write; `ICANON=0x2, ECHO=0x8, ISIG=0x1, TCSANOW=0, VMIN=6, VTIME=5`.
  - Raw mode guard: orig termios in `OnceLock<Termios>`; restore + `\x1b[?1049l\x1b[?25h\x1b[0m` on exit AND in a panic hook.
  - Pure encoder: `pub(crate) fn frame_bytes(cells: &[Cell], prev: Option<&[Cell]>, ascii: bool) -> Vec<u8>` — full frame (`\x1b[2J\x1b[H` + all cells) when prev None; else escapes only for changed cells. Track cursor + last fg/bg to minimize escapes. `fn c256(rgb: u32) -> u8` (6×6×6 cube + grayscale ramp for near-gray).
  - `ascii` maps u16 glyphs ≥128 → ASCII fallback ('#', '=', etc.).
  - Input: blocking 1-byte read; ESC → VTIME=1/VMIN=0 follow-up read (arrows `[A-D`, F5 `[15~`, F1 `OP`/`[11~`); lone ESC = quit; q quit, wasd/hjkl/arrows move, `.` wait, r restart, F5 save (double-press confirm), F1 world info.
  - `pub(crate) fn run(...)` same signature as minifb backend; also `pub(crate) fn render_frame_main(seed: u64, ascii: bool)` for `--render-frame`.
- Modify: `src/main.rs`: `--render-frame` + `--ascii` flags (cfg backend-term), dispatch.
- Create: `tests/golden/frame_seed_1.bin`, `frame_seed_42.bin`, `frame_seed_1_ascii.bin` (captured; RE-CAPTURED after task 5).
- Modify: `Makefile`: `test-term` (`cargo test --no-default-features --features backend-term --quiet`), `frames` target (term build → `--render-frame --seed N` vs fixtures), wire into check. Term builds use `--target-dir target/term` so the default build isn't clobbered.

**Tests (in backend_term.rs `#[cfg(test)]`):** c256 vectors (0x000000→16-ish/232, 0xFFFFFF→231, pure red→196); frame_bytes dirty-diff (change one cell → output contains exactly one cursor move); ascii fallback total (no bytes ≥ 0x80 in ascii mode output).

- [x] Steps: implement; capture frame goldens; `make check` green. Interactive terminal feel → playtest-pending. Commit: `feat: ANSI terminal backend (backend-term) + --render-frame goldens`.

### Task 5: Aesthetic pass

**Sub-items:**
